## test4.c
### 1. 字符串字面量 (String Literal) —— “博物馆里的受保护文物”

**长什么样：**
凡是你直接在代码里用双引号 `""` 括起来写死的内容，统统都是字符串字面量。

```c
char *s = "Hello"; // "Hello" 就是字面量，s 只是指向它的指针

```

**它的宿命：**

* **只读（Read-Only）：** 编译器在编译时，会把 `"Hello"` 连同末尾的 `\0` 一起，打包塞进程序内存里的一个特殊区域，叫作**只读数据段 (Read-Only Data Segment)**。
* **绝对不可修改：** 因为它在只读区，操作系统对这块内存是严加看管的。如果你敢在代码里写 `s[0] = 'h';` 试图把大写的 H 改成小写，编译时不会报错，但**程序一运行，立马崩溃**（也就是大名鼎鼎的 `Segmentation fault` 段错误）。
* **与天地同寿：** 只要你的程序还在运行，这个字面量就一直躺在那个内存里，直到程序完全结束才会被销毁。

---

### 2. 不是字面量的字符串 (字符数组) —— “你自己的草稿本”

**长什么样：**
通常是我们自己申请的一块内存（比如数组），然后在里面装入了字符。

```c
char arr[] = "Hello"; // arr 是一个真实的数组（非字面量）

```

**它的宿命：**

* **可以随意涂改：** 这里的 `arr` 是在内存的**栈区 (Stack)** 里为你专门开辟的一块包含 6 个字节的空间。
* **它是被“复制”过来的：** 编译器看到这行代码时，会先在只读区弄一个 `"Hello\0"`，然后在程序运行到这一行时，把这 6 个字符**复制一份**，塞进你的 `arr` 数组里。
* **随你修改：** 因为 `arr` 是你自己的“草稿本”，所以你完全可以写 `arr[0] = 'h';`，程序不仅不会崩溃，而且还会成功把数组里的内容变成 `hello\0`。

---

### 3. 一张表总结核心区别

| 特性 | `char *s = "Hello";` (指针指向字面量) | `char arr[] = "Hello";` (普通的字符数组) |
| --- | --- | --- |
| **内存位置** | **只读数据段** (Read-Only Data) | **栈区** (Stack) 或 堆区 (Heap) |
| **能否修改** | ❌ **绝对不能！** 一改就崩溃 | ✅ **可以随便改！** |
| **指针含义** | `s` 只是一个路牌，指向上面的受保护区域 | `arr` 是这块内存的真实主人 |
| **本质操作** | 直接把指针指向已有的“文物” | 在你的地盘上“复印”了一份数据 |

---

**一句话总结：**
带 `*` 直接等于双引号的，是只读的字面量，只能看不能摸；带 `[]` 的数组，是复制到你手里的副本，想怎么改就怎么改。
